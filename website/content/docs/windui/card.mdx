`Card` is a visual navigation element. The primary pattern is pressing the
card itself to open a dedicated page, similar to selecting a tab.

It can still hold normal WindUI elements when you need a compact dashboard
inside the card. `HStack`, `VStack`, `Button`, `Toggle`, and the rest of the
element API remain available.

## Page Card

```luau
local Card = Tab:Card({
    Title = "Card Navigation",
    Desc = "Tap the card to open a dedicated page.",
    Icon = "panels-top-left",
    Color = "#0EA5E9",
    OpenTab = true,
    TabTitle = "Card Detail",
    TabIcon = "panels-top-left",
    Build = function(CardPage)
        CardPage:Callout({
            Title = "Card Page",
            Desc = "This page was opened by pressing the card.",
            Variant = "Success",
        })
    end,
})
```

## Nested Content

Nested content is optional. Use it when the card needs live state or compact
controls before the user opens the full card page.

```luau
Card:Button({
    Title = "Normal element inside card",
    Icon = "mouse-pointer-click",
})

local Row = Card:HStack({
    MinChildWidth = 140,
})

Row:Badge({
    Title = "State",
    Value = "Ready",
    Variant = "Success",
})
```

## CardButton

`CardButton` adds a secondary action inside the card without changing the
main press behaviour.

```luau
Card:CardButton({
    Title = "Run action",
    Icon = "bell",
    Callback = function()
        print("card action")
    end,
})
```

## CardTab

`CardTab` is kept for compatibility and secondary actions. Prefer
`OpenTab = true` on the card itself when the whole card should navigate.

```luau
Card:CardTab({
    Title = "Open details",
    Icon = "arrow-right",
    TabTitle = "Card Detail",
    TabIcon = "panels-top-left",
    Build = function(DetailTab)
        DetailTab:Callout({
            Title = "CardTab Page",
            Desc = "This page was created from a card.",
            Variant = "Success",
        })
    end,
})
```
